# Evaluation

The evaluation harness proves three properties of the memory system:

1. Support continuity improves when cross-session memory is enabled (re-ask rate drops to 0).
2. Superseded facts are replaced, not accumulated (stale data cannot surface).
3. Expired facts are forgotten after their TTL (time-gated forgetting works).

## Command

```bash
pnpm eval
```

## Scenarios

### Scenario 1 — Basic Recall (Acme Corp)

The same synthetic support scenario is replayed twice:

| Mode | Behavior |
|---|---|
| Memory ON | Semantic and episodic cross-session retrieval are enabled. |
| Memory OFF | Semantic and episodic cross-session retrieval are disabled; current-session working memory remains enabled. |

Keeping current-session working memory enabled makes the baseline fair. The comparison is about cross-session continuity, not whether the agent can remember a message from two turns ago.

**Assertions:** `memoryOnReaskRate === 0`, `memoryOffReaskRate === 1`.

### Scenario 2 — Supersession (Globex Corp)

A technical contact fact is written in session 1 (`Amanda → technical_contact`), then superseded in session 2 (`Bob → technical_contact`). After close and distillation:

- The stale Amanda fact must no longer appear in recall results.
- The current Bob fact must appear.

**Assertions:** `supersededFactGone === true`, `updatedFactPresent === true`.

### Scenario 3 — TTL Forgetting (Initech)

A fact is written with `ttlDays: 1` during a session closed at `T`. Recall is performed at `T + 3 days`. The expired fact must not appear in results.

**Assertions:** `expiredFactGone === true`.

## Reproducibility

The eval path is deterministic. It uses:

- synthetic fixtures with distinct seed ranges per scenario (21–27, 31–36, 41–43),
- a fake Qwen client for stable embeddings and fact extraction,
- fixed assertions that throw on regression.

The fake Qwen client is intentional. It keeps judgments reproducible without requiring a live API key. The live API path uses Qwen Cloud when `DASHSCOPE_API_KEY` is configured.

## Anti-cheat guardrails

- Ground truth is separate from the conversation fixture.
- Ground truth is not shown to the agent.
- Memory OFF disables only cross-session retrieval.
- Hallucinated facts are counted, not ignored.
- Supersession and TTL assertions check actual DB state, not just agent output.
- The output JSON is generated by the harness, not hand-written.
