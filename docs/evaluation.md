# Evaluation

The evaluation harness proves the core claim: support continuity improves when cross-session memory is enabled.

## Command

```bash
pnpm eval
```

## Metrics

The harness reports:

- **re-ask rate:** whether Nat asks the customer to repeat already-known context,
- **context recall accuracy:** whether the required facts are present in the response path,
- **hallucination count:** whether Nat invents unsupported facts.

## Ablation

The same synthetic scenario is replayed twice:

| Mode | Behavior |
|---|---|
| Memory ON | Semantic and episodic cross-session retrieval are enabled. |
| Memory OFF | Semantic and episodic cross-session retrieval are disabled; current-session working memory remains enabled. |

Keeping current-session working memory enabled makes the baseline fair. The comparison is about cross-session continuity, not whether the agent can remember a message from two turns ago.

## Reproducibility

The eval path is deterministic. It uses:

- synthetic fixtures,
- hidden ground truth in `eval/ground_truth.json`,
- a fake Qwen client for stable embeddings and fact extraction,
- fixed assertions around the expected output.

The fake Qwen client is intentional. It keeps the judged number reproducible without requiring a live API key. The live API path uses Qwen Cloud when `DASHSCOPE_API_KEY` is configured.

## Expected output

```text
memory-on re-ask rate: 0.00
memory-on recall accuracy: 1.00
memory-on hallucination count: 0
memory-off re-ask rate: 1.00
memory-off recall accuracy: 0.00
memory-off hallucination count: 0
re-ask rate: 0.00 (memory) vs 1.00 (no-memory)
```

## Anti-cheat guardrails

- Ground truth is separate from the conversation fixture.
- Ground truth is not shown to the agent.
- Memory OFF disables only cross-session retrieval.
- Hallucinated facts are counted, not ignored.
- The output line is generated by the harness, not hand-written in the README.
